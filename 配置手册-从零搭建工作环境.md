# 从零搭建工作环境 - 配置手册

> 本手册记录了在公司和家用电脑之间同步 Obsidian 笔记的完整配置流程。
> GitHub 仓库: https://github.com/shenjiangshuai3-creator/obsidian-vault

---

## 一、需要安装的软件

### 1. VS Code
- 下载: https://code.visualstudio.com/
- 安装：一路默认 Next
- 安装插件：Continue（左栏插件商店搜索 "Continue" 安装）

### 2. Obsidian
- 下载: https://obsidian.md/
- 安装：一路默认 Next

### 3. Python 3.13.x
- 下载: https://www.python.org/downloads/
- 安装时 **务必勾选** "Add Python to PATH"
- 验证：打开终端运行 `python --version`，应显示 Python 3.13.x

### 4. Git
- 下载: https://git-scm.com/downloads
- 安装：一路默认 Next
- 验证：打开终端运行 `git --version`，应显示 git version 2.x.x

---

## 二、配置 Git 身份信息

打开终端（CMD 或 PowerShell），依次执行：

```powershell
git config --global user.name "shenjiangshuai3-creator"
git config --global user.email "shenjiangshuai3@gmail.com"
```

## 三、配置 Git 代理（如果用了科学上网工具）

ikuuuvpn/Clash 等工具的端口通常是 7890：

```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

> 如果用的其他代理工具，在任务栏右下角找到代理图标，查看 HTTP 代理端口是多少，替换上面的 7890

## 四、从 GitHub 克隆笔记到家

### 1. 创建 Obsidian Vault
打开 Obsidian -> 在电脑上创建一个新空文件夹作为 Vault：
```
C:\Users\你的用户名\Documents\Obsidian Vault
```

### 2. 克隆仓库
打开终端（CMD 或 PowerShell）：

```powershell
cd C:\Users\你的用户名\Documents
git clone https://github.com/shenjiangshuai3-creator/obsidian-vault.git "Obsidian Vault"
```

### 3. 在 Obsidian 中打开
- Obsidian -> "打开本地文件夹" -> 选择 C:\Users\你的用户名\Documents\Obsidian Vault
- 之后就能看到 Python学习笔记 和 SQL笔记 两个目录了

## 五、日常同步笔记

每次在 Obsidian 写完笔记后，同步到 GitHub：

```powershell
cd C:\Users\你的用户名\Documents\"Obsidian Vault"
git add .
git commit -m "更新笔记"
git push
```

或者双击 Vault 目录下的 **sync_notes.bat** 一键同步。

### 家用电脑同步公司最新内容：

```powershell
cd C:\Users\你的用户名\Documents\"Obsidian Vault"
git pull
```

## 七、常见问题

### Q: git push 报错 Connection reset 或 Could not connect
**原因**：网络需要代理才能访问 GitHub
**解决**：
```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

### Q: python 命令找不到
**原因**：安装 Python 时没勾选 "Add Python to PATH"
**解决**：重装 Python，或手动添加环境变量

### Q: 家用和公司同步时冲突
**解决**：
```powershell
git pull --rebase
# 如果有冲突，解决后：
git add .
git rebase --continue
git push
```
