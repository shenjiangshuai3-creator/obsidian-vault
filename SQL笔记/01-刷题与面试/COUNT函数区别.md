# COUNT函数区别

> 数据开发面试高频题，也是实际写 SQL 容易混淆的地方

---

## 一、结论先行

| 写法 | 含义 | 性能 |
|------|------|------|
| `COUNT(*)` | 统计**所有行数**，包括 NULL 值行 | ✅ 优化器识别最优，**推荐使用** |
| `COUNT(1)` | 统计**所有行数**，效果等价于 COUNT(*) | ✅ 与 COUNT(*) 几乎无差别 |
| `COUNT(列名)` | 统计该列**非 NULL 值的行数** | ⚠️ 功能不同，不要混淆 |
| `COUNT(DISTINCT 列名)` | 统计该列**去重后的非 NULL 值个数** | ⚠️ 功能不同 |

**结论：`COUNT(1)` 和 `COUNT(*)` 没有区别，结果一样，性能一样，随便用哪个都行。**

---

## 二、常见误解

### ❌ 误解1：COUNT(1) 比 COUNT(*) 快

**真相：** 现在的主流数据库（MaxCompute、MySQL 8.0+、PostgreSQL、SQL Server）都会把 `COUNT(1)` 优化成 `COUNT(*)`，性能完全一样。

### ❌ 误解2：COUNT(*) 会读取所有列，性能差

**真相：** `COUNT(*)` 不会读取具体数据，数据库会用**最小代价**来统计行数，比如走二级索引。

### ❌ 误解3：COUNT(1) 只统计第一列

**真相：** `1` 只是个常量，跟列没有任何关系，它统计的是行数。

## 三、实际验证

假设 `users` 表有 5 行数据，其中 `email` 字段有 2 个 NULL：

| id | name | email |
|----|------|-------|
| 1 | 张三 | z@.com |
| 2 | 李四 | l@.com |
| 3 | 王五 | w@.com |
| 4 | 赵六 | NULL |
| 5 | 钱七 | NULL |

```sql
SELECT COUNT(*)    FROM users;  -- 结果: 5（所有行）
SELECT COUNT(1)    FROM users;  -- 结果: 5（所有行）
SELECT COUNT(id)   FROM users;  -- 结果: 5（id 没有 NULL）
SELECT COUNT(email) FROM users; -- 结果: 3（email 只有 3 个非 NULL）
```

## 四、建议

| 场景 | 推荐写法 |
|------|---------|
| 数总行数 | `COUNT(*)` 或 `COUNT(1)` 都可以 |
| 数非空值数量 | `COUNT(列名)` |
| 数去重数量 | `COUNT(DISTINCT 列名)` |
| 分组后计数 | `COUNT(*)`（搭配 GROUP BY） |

### 个人建议

> 统一用 `COUNT(*)`，理由：
> 1. 语义最清晰 —— "数行数"
> 2. 业界标准写法
> 3. 避免别人误解

---

## 五、面试回答模板

> **面试官：** COUNT(1) 和 COUNT(*) 有什么区别？
>
> **回答：** 功能上没有区别，都是统计总行数。性能上也没有区别，现代数据库优化器会把 COUNT(1) 优化成 COUNT(*) 相同的执行计划。我个人习惯用 COUNT(*)，语义更清晰。
