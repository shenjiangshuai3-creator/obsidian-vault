# SQL 笔记总览

> 数据开发人员专属的 SQL 笔记体系
> 包含：笔记目录、使用指南、体系建设建议

---

## 一、笔记目录结构

```
SQL笔记/
├── 01-刷题与面试       ← 日常刷题 + 面试准备
├── 02-踩坑记录         ← 工作中遇到的问题和解决方案 ⭐
├── 03-常用脚本库        ← 你经常用的 SQL 模板（直接复制用） ⭐
├── 04-ER图             ← 所有 ER 图集中管理
├── 05-数据探查          ← 所有数据探查脚本
├── 06-指标与口径        ← 业务指标的定义和计算逻辑
├── 07-函数与技巧        ← 常用函数、窗口函数、优化技巧
├── 08-学习笔记          ← 系统学习时记的笔记
└── SQL笔记总览.md       ← 本文件（总入口）
```

### 为什么这样分？

| 板块 | 定位 | 使用场景 |
|------|------|---------|
| **01-刷题与面试** | 刷 LeetCode / 牛客的 SQL 题 | 碎片时间刷题，面试前复习 |
| **02-踩坑记录** ⭐ | 工作中遇到的 Bug 和解决方案 | 遇到同类问题直接查，不用再百度 |
| **03-常用脚本库** ⭐ | 开箱即用的 SQL 模板 | 写代码时直接复制改参数 |
| **04-ER图** | 所有表关系图集中管理 | 接手新需求时看表结构 |
| **05-数据探查** | 各表的数据探查脚本 | 数据质量问题排查 |
| **06-指标与口径** | GMV、完播率等口径定义 | 避免不同人理解不一致 |
| **07-函数与技巧** | 窗口函数、优化技巧 | 写复杂 SQL 时参考 |
| **08-学习笔记** | 新学内容 | 系统提升时用 |

## 三、推荐工作流

### 每天下班前花 5 分钟

```
1. 把今天写的复杂 SQL 存入 03-常用脚本库
2. 把今天踩的坑存入 02-踩坑记录
3. 把今天接触的新表关系存入 04-ER图
4. git push 同步到 GitHub
```

### 三个月后的收益

| 时间 | 积累成果 |
|------|---------|
| 1 个月 | 10+ 个踩坑记录，5+ 个常用脚本 |
| 3 个月 | 30+ 踩坑记录，20+ 脚本模板，覆盖大部分工作场景 |
| 6 个月 | 形成个人知识库，遇到问题 80% 直接查笔记解决 |

## 四、AI 使用技巧

### 推荐的提问模板

| 场景 | 提问方式 |
|------|---------|
| 创建笔记 | "帮我在 Obsidian 中创建一篇笔记，标题是xxx，存入 SQL笔记/03-常用脚本库" |
| 生成 ER 图 | "根据这段 SQL 生成 ER 图，连线标注 字段=字段 的格式，存入 SQL笔记/04-ER图" |
| 数据探查 | "帮我探查 xxx 表的数据质量，存入 SQL笔记/05-数据探查" |
| 踩坑记录 | "帮我整理 xxx 问题的踩坑记录，存入 SQL笔记/02-踩坑记录" |
| 修改文件 | "帮我把 xxx 文件中的 xxx 修改为 xxx，保留我之前的个性化修改" |

### 注意事项

| 原则 | 说明 |
|------|------|
| ✅ **明确指定路径** | 告诉 AI 文件存在哪个目录 |
| ✅ **提供完整 SQL** | ER 图和探查结果更准确 |
| ✅ **标注个性化修改** | 在笔记中开辟记录区域，避免被覆盖 |
| ✅ **关闭 Accept 确认** | `continue.enableAcceptConfirmation = false` |
