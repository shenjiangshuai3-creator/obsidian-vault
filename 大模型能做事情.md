__1. 表资产理解和梳理__

- 读取 MaxCompute 表结构、字段、分区、注释
- 根据表名、字段名、业务线资料判断表大概属于哪个主题域
- 帮你整理 ODS、DWD、DWS、ADS 分层关系
- 根据 `业务线_数仓表映射-1.md`、`taidou_local_life_tables.txt` 这类文件生成表清单、主题域映射、血缘草图

__2. SQL 编写和校验__

- 根据中文需求生成 MaxCompute SQL
- 自动补充分区条件、时间条件、聚合口径
- 帮你检查 SQL 语法、字段是否存在、表是否存在
- 优化慢 SQL，比如减少全表扫、改写 JOIN、调整分区过滤
- 把临时分析 SQL 改成可沉淀的 DWD/DWS/ADS SQL

__3. 指标口径建设__

- 把“项目数、线索数、转化率、履约率、GMV、核销率”这类业务指标拆成明确口径
- 输出指标定义：指标名称、业务含义、统计粒度、过滤条件、时间口径、来源表、SQL 示例
- 检查不同报表之间口径不一致的问题
- 帮你设计指标字典或指标管理表

__4. ODS 到数仓建模__

- 从业务系统表、飞书表、接口 JSON 里识别实体和关系
- 设计 ODS 清洗规则、DWD 明细事实表、维表、DWS 汇总表、ADS 应用表
- 给出建表 DDL、字段命名、分区设计、生命周期建议
- 帮你把“业务线 -> 数据源表 -> 数仓表 -> 报表”串起来

__5. 数据质量检查__

- 生成空值率、重复率、主键唯一性、枚举值异常、分区波动检查 SQL
- 对比源表和目标表条数、金额、状态分布
- 设计日常巡检 SQL 或质量规则清单
- 帮你定位“今天数据少了/字段为空/状态对不上”的原因

__6. 文档和知识库沉淀__

- 把已有 SQL、表结构、业务说明整理成 Markdown 知识库
- 自动生成表说明文档、字段说明文档、指标说明文档
- 把散落在飞书、Obsidian、Markdown 里的资料整理成可检索结构
- 生成“数仓表使用指南”和“常用分析 SQL 模板”

__7. 自动化 Agent 能力__

现在已经创建了 `maxcompute_sql_agent.py`，后面可以继续增强成一个本地数仓助手：

- 输入中文问题，自动找表、查字段、生成 SQL
- 连接 MaxCompute 执行 `EXPLAIN` 校验
- 读取本地表映射文件辅助选表
- 输出 SQL + 口径说明 + 风险点
- 以后还可以加：自动跑查询、导出 CSV、生成日报、同步飞书文档
